站长操作流程

图片预览更换与正式上传流程

这个页面说明怎样安全测试首页图片。你可以先在浏览器里临时替换图片，确认效果后，再把最终图片正式加入 GitHub，并通过 Vercel 预览部署检查。

1. 打开编辑预览模式

在首页地址后加 ?edit=1。例如：/zh/?edit=1 或英文首页 /?edit=1。

2. 本地替换图片

点击可以编辑的图片，或把图片拖到图片区域。替换只保存在你自己的浏览器里，用于预览效果。

3. 检查视觉效果

分别看桌面和手机布局。建议图片小于 1.5MB，这样浏览器更容易保存本地预览。

4. 导出更换清单

点击工具条里的导出更换清单。把下载的 JSON 文件发给 ChatGPT，或把内容贴过来，我就能知道每张图对应哪个位置。

5. 正式上传图片

可以使用编辑工具条里的 GitHub 上传入口，或把最终选中的图片交给我，由我给出文件命名和放置建议。

6. 提交后再次预览

图片文件真正提交到分支后，Vercel 会自动生成新的 Preview Deployment，再进行最后检查。

重要说明：这个编辑模式不是正式 CMS，也不会把图片永久上传到 Vercel。它只是一个安全的站长预览工具，用来先选图、试图、确认效果。
