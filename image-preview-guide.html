Image Preview and Final Upload Workflow

This page explains how to test homepage images safely before they are permanently added to the GitHub repository and deployed to Vercel.

1. Open preview edit mode

Open the homepage with ?edit=1. Example: /?edit=1 or /zh/?edit=1.

2. Replace images locally

Click an editable image, or drag an image file onto it. The replacement is saved only in your own browser for preview.

3. Check visual fit

Review desktop and mobile layout. Keep images under about 1.5MB for reliable local preview.

4. Export the change list

Click Export change list. Send the JSON file or paste its contents into ChatGPT so the final image slots are clear.

5. Upload final images

Use the GitHub upload link in the edit toolbar, or share the chosen files with ChatGPT for final naming and placement guidance.

6. Commit and preview again

After real image files are committed to the branch, Vercel will create a new Preview Deployment for final checking.

Important: The edit mode is not a real CMS. It does not permanently upload images to Vercel. It is a safe owner-only preview tool for choosing final images before committing them to GitHub.
